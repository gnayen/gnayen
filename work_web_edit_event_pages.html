Accupass 活動編輯

Web ui / ux design

主要痛點

1. 活動內容 ”編輯器” 難以編輯

2. 整體編輯活動時程太長 ( 約 40 分鐘 ~ 60分鐘 ）

3. 一頁打好幾支 API Loading 太重，Bug 多

對應解法

1. 活動內容編輯器更換套件，編輯器工具列 fixed on top 不用來回點按工具編輯

2. 全程使用無腦編輯，以降低編輯時程，目標希望可以壓至 20 分鐘以內 ( ex. 提供模版供主辦填寫，自動代入建議的 value，刪除不必須的功能 )

3. 把原本一頁式的編輯活動頁拆成好幾個 steps，一頁只打一隻 API Loading 相對減輕
